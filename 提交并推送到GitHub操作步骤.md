# 提交并推送到GitHub操作步骤

## 快速参考

### 1. 基本提交流程

```bash
# 1. 查看当前状态
git status

# 2. 添加指定文件到暂存区
git add README.md

# 或添加所有修改的文件到暂存区
git add .

# 3. 提交到本地仓库
git commit -m "提交描述信息"

# 4. 推送到GitHub（第一次需要设置上游分支）
git push --set-upstream origin master

# 后续推送直接使用
git push
```

### 2. 详细操作步骤

#### 2.1 检查修改状态

在提交前，先查看当前项目的修改状态：

```bash
git status
```

输出会显示：
- 已修改但未添加到暂存区的文件
- 已添加到暂存区但未提交的文件
- 未跟踪的文件（新建文件）

#### 2.2 添加文件到暂存区

选择要提交的文件：

```bash
# 添加单个文件
git add 文件名

# 添加多个文件
git add 文件1 文件2 文件3

# 添加所有修改的文件
git add .

# 添加所有修改的Python文件
git add *.py

# 添加指定目录下的所有文件
git add 目录名/
```

#### 2.3 提交到本地仓库

使用清晰的提交信息提交修改：

```bash
git commit -m "提交描述信息"
```

提交信息建议遵循以下规范：
- 简短明了，不超过50个字符
- 第一个单词大写，使用祈使句
- 说明具体修改内容，例如：
  - `修复登录功能的bug`
  - `添加用户管理模块`
  - `更新README文档`
  - `版本号更新为1.0.1`

#### 2.4 推送到GitHub

##### 第一次推送

如果是第一次推送，需要设置上游分支：

```bash
git push --set-upstream origin master
```

或简写为：

```bash
git push -u origin master
```

这个命令会：
- 将本地master分支推送到远程origin仓库
- 设置本地master分支跟踪远程master分支

##### 后续推送

设置好上游分支后，后续推送可以直接使用：

```bash
git push
```

#### 2.5 推送指定分支

如果需要推送其他分支：

```bash
git push origin 分支名
```

### 3. 常见场景处理

#### 3.1 推送失败的处理

如果推送失败，可能是以下原因：

1. **未设置上游分支**
   ```bash
   git push --set-upstream origin 分支名
   ```

2. **远程仓库有新的提交**
   ```bash
   # 先拉取远程最新代码
   git pull
   # 解决可能的冲突后再推送
   git push
   ```

3. **认证失败**
   - 检查GitHub账号是否正确登录
   - 检查远程仓库URL是否正确
   - 重新配置Git凭证

#### 3.2 查看推送状态

```bash
# 查看远程分支状态
git branch -av

# 查看最近的推送记录
git log --oneline -n 10
```

#### 3.3 推送标签

如果创建了版本标签，可以推送标签到远程仓库：

```bash
# 推送单个标签
git push origin v1.0.1

# 推送所有标签
git push origin --tags
```

### 4. 最佳实践

1. **频繁提交**：完成一个小功能或修复一个bug后立即提交
2. **清晰的提交信息**：便于后续查看和回滚
3. **定期推送**：每天至少推送一次代码到远程仓库
4. **分支管理**：开发新功能时创建新分支，完成后合并到主分支
5. **使用标签**：发布新版本时创建标签，便于版本管理
6. **先拉取后推送**：推送前先拉取远程最新代码，避免冲突

### 5. 常用命令速查表

| 命令 | 功能 |
|------|------|
| `git status` | 查看当前状态 |
| `git add .` | 添加所有修改的文件到暂存区 |
| `git commit -m "信息"` | 提交到本地仓库 |
| `git push` | 推送到远程仓库 |
| `git pull` | 拉取远程最新代码 |
| `git log` | 查看提交历史 |
| `git branch -a` | 查看所有分支 |
| `git checkout 分支名` | 切换分支 |
| `git tag v1.0.1` | 创建标签 |
| `git push --tags` | 推送所有标签 |

---

**注意**：在执行推送操作前，请确保：
1. 已关联正确的远程仓库
2. 已设置正确的Git全局配置
3. 已添加所有需要提交的文件
4. 提交信息清晰明了

以上就是提交并推送到GitHub的完整操作步骤，根据实际项目需求可以灵活调整。
