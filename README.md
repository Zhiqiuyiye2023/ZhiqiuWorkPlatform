## 📖 项目简介

知秋工作平台是一款功能丰富的桌面应用程序，集成了数据处理、文件转换、影像处理、AI功能等多种实用工具。采用现代化的 Fluent Design 风格，提供优雅的用户体验和强大的功能模块。

### ✨ 核心亮点

- 🎨 **现代化 UI** - Microsoft Fluent Design 设计语言
- 🌓 **主题系统** - 浅色/深色主题，自定义主题色
- 🚀 **模块化架构** - 多个独立功能模块，易于扩展
- 🔍 **全局搜索** - 任意页面快速查找功能
- 💾 **数据持久化** - 自动保存配置和使用历史
- ⚙️ **可视化设置** - 无需编辑代码即可配置
- 🤖 **AI功能集成** - YOLO工具箱支持目标检测、实例分割等
- 🌍 **GIS工作流** - 拖拽式GIS数据处理工作流

## 🚀 快速开始

### 方法一：一键启动（推荐）
```batch
双击 "启动应用.bat" 文件
```

### 方法二：命令行启动
```bash
cd "c:\知秋工作平台"
python demo.py
```

## 📋 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Python | 3.8 - 3.11 (推荐 3.9 或 3.10) |
| 内存 | 至少 4GB RAM |
| 磁盘空间 | 至少 500MB 可用空间 |
| 核心依赖 | PyQt6 >= 6.7.0, PyQt6-Fluent-Widgets >= 1.1.0, PyQt6-Frameless-Window >= 0.4.0 |

## 📦 环境检查与安装

### 🔍 检查环境

**方式 1: 使用环境检查脚本（推荐）**
```bash
python check_env.py
```
这将检查所有依赖包的安装状态和版本。

**方式 2: 手动检查**
```bash
pip list | findstr "PyQt6"
```

### 📥 安装依赖

**方式 1: 快速安装核心依赖（推荐新手）**
```bash
双击运行 "安装核心依赖.bat"
```
或命令行：
```bash
pip install PyQt6>=6.7.0 PyQt6-Fluent-Widgets>=1.1.0 PyQt6-Frameless-Window>=0.4.0 darkdetect>=0.8.0 requests>=2.31.0
```

**方式 2: 安装完整依赖**
```bash
pip install -r requirements.txt
```

**方式 3: 分模块安装**
- 安装GIS功能依赖：双击 `安装GIS依赖.bat`
- 安装文件处理依赖：双击 `安装文件处理依赖.bat`
- 安装AI功能依赖：双击 `安装AI依赖.bat`（用于YOLO工具箱）

### ⚠️ 安装注意事项

1. **国内用户加速**：如果安装速度慢，可使用清华镜像源：
   ```bash
   pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
   ```

2. **GIS依赖安装失败**：推荐使用conda安装
   ```bash
   conda install -c conda-forge geopandas
   ```

3. **详细安装说明**：查看 [`环境配置说明.md`](环境配置说明.md) 获取完整的安装指南

## 🌍 GIS工作流功能详解

### 🎯 功能概述

GIS工作流功能提供了一个类似于FME的拖拽式GIS数据处理环境，用户可以通过可视化的方式构建复杂的数据处理流程。该功能现在作为主界面的一个独立模块，与应用、视频、YOLO工具箱等平级显示。

### 🧩 核心特性

- **拖拽式操作**：从模块库中拖拽功能模块到工作区
- **可视化流程设计**：直观地构建数据处理流程
- **模块属性配置**：双击模块可配置详细参数
- **丰富的GIS功能**：涵盖数据输入、分析、处理和输出全流程

### 📦 功能模块

#### 添加数据
- **添加文件**：支持多种GIS数据格式的导入

#### 分析功能
- **相交**：计算两个图层的交集
- **擦除**：从一个图层中移除与另一个图层重叠的部分
- **标识**：保留输入要素并在重叠区域添加标识属性

#### 数据处理
- **融合**：根据指定字段合并相似要素
- **合并**：将多个图层合并为一个图层

#### 导出数据
- **导出Excel**：将处理结果导出为Excel格式
- **导出SHP**：将处理结果导出为Shapefile格式

### 🎨 界面布局

1. **左侧模块栏**：包含所有可用的功能模块，按类别组织
2. **中间工作区**：拖拽模块到此处构建工作流
3. **右侧属性面板**：配置选中模块的参数

### 🚀 使用流程

1. 从左侧模块栏中选择需要的功能模块
2. 将模块拖拽到中间工作区
3. 双击模块打开属性面板进行参数配置
4. 连接模块形成数据处理流程
5. 点击"运行工作流"执行处理

## 🤖 自动化工具详解

### 🎯 功能概述

自动化工具是一个基于DrissionPage的模块化浏览器自动化工具，支持表格数据驱动的自动化操作。该工具允许用户配置多个元素操作模块，实现复杂的浏览器自动化工作流。

### 🧩 核心特性

- **模块化设计**：将自动化流程拆分为多个独立的元素操作模块
- **表格数据驱动**：支持从Excel文件读取数据，实现批量自动化操作
- **可视化配置**：直观的UI界面，无需编写代码即可配置自动化流程
- **灵活的元素交互**：支持多种元素定位方式和交互动作
- **条件执行**：支持配置条件，根据条件决定是否执行特定操作
- **主题适配**：完全支持浅色/深色主题切换

### 📦 功能模块

#### 核心组件
- **浏览器连接器**：处理与Chrome浏览器的连接和通信
- **表格数据管理器**：读取和管理Excel表格数据
- **元素模块**：定义单个元素的操作配置
- **元素模块管理器**：管理多个元素操作模块
- **自动化流程**：协调各模块执行自动化工作流

### 🎨 界面布局

1. **左侧控制面板**：包含数据加载、浏览器连接和流程控制
2. **右侧配置区域**：配置元素操作模块和执行参数
3. **主题适配**：支持浅色/深色主题自动切换

### 🚀 使用流程

1. 启动Chrome浏览器并开启调试模式
2. 在工具中连接到浏览器
3. 加载Excel表格数据
4. 配置元素操作模块（定位方式、交互动作、等待时间等）
5. 设置执行条件（可选）
6. 运行自动化流程
7. 查看执行结果

## 🤖 YOLO工具箱详解

### 📊 数据处理模块

处理GIS矢量数据以创建YOLO训练数据集：

- **矢量数据处理**：将SHP文件转换为YOLO格式的标注文件
- **影像裁剪**：根据矢量要素裁剪影像生成训练样本
- **数据集划分**：自动划分训练集和验证集
- **类别映射**：支持自定义类别映射和数据集合并
- **多种格式支持**：支持目标检测和实例分割数据格式

### 🏋️ 模型训练模块

训练和微调YOLO模型：

- **模型选择**：支持YOLOv8系列模型（n/s/m/l/x）
- **参数配置**：可调节epochs、batch size、学习率等超参数
- **设备选择**：支持CPU和GPU训练（推荐GPU）
- **进度监控**：实时显示训练进度、损失曲线和mAP指标
- **断点续训**：支持从上次中断处继续训练
- **模型验证**：训练完成后自动验证模型性能

### ✏️ 实例分割标注模块

创建实例分割标注数据：

- **交互式标注**：通过点击添加多边形节点进行标注
- **类别管理**：支持自定义类别和颜色
- **标注编辑**：可修改、删除已有标注
- **快捷键支持**：支持常用操作快捷键
- **数据保存**：自动保存标注数据为YOLO格式
- **批量处理**：支持批量标注多张图片

### 🔍 影像检测模块

多种目标检测功能：

#### 实时检测
- **摄像头检测**：使用摄像头进行实时目标检测
- **屏幕捕获**：捕获屏幕指定区域进行检测
- **窗口捕获**：选择特定窗口进行检测

#### 本地影像检测
- **图片检测**：对单张图片进行目标检测
- **视频检测**：对视频文件进行目标检测
- **结果展示**：可视化检测结果，支持缩放和拖拽

#### 遥感影像识别 ⭐
- **大影像处理**：支持处理大型遥感影像（>10000x10000像素）
- **智能切片**：自动将大影像切分为小块进行处理
- **结果合并**：自动合并切片检测结果
- **矢量输出**：生成检测结果的矢量文件（SHP格式）
- **参数调节**：可调节置信度、IoU阈值等参数
- **GPU加速**：支持CUDA GPU加速（强烈推荐）

#### ⚠️ 遥感影像识别注意事项
由于YOLO模型在CPU模式下的内存限制，处理大型遥感影像时可能出现栈溢出崩溃（错误代码0xC0000409）。为确保稳定运行，建议：

1. **使用GPU模式**（推荐）：安装CUDA支持的PyTorch版本
2. **降低参数**：使用较小的模型（如yolov8s-seg.pt）和切片大小（512）
3. **影像预处理**：先将大影像裁剪为较小的块再处理
4. **分批处理**：避免一次性处理过多影像

详细解决方案请查看 [`遥感影像识别CPU模式崩溃临时解决方案.md`](docs/fixes/遥感影像识别CPU模式崩溃临时解决方案.md)

## ⚠️ 注意事项

### 1. Mica 效果
- ✅ **Windows 11**: 完全支持
- ❌ **Windows 10**: 不支持（选项禁用）

### 2. 亚克力效果
当前使用基础版，亚克力效果可能无法完全显示。如需完整支持：
```bash
pip install PyQt6-Fluent-Widgets[full]
```

### 3. 重启生效
以下设置需要重启应用：
- 界面缩放
- 语言设置

### 4. AI功能依赖
YOLO工具箱需要安装额外的AI依赖包：
```bash
# 安装AI功能依赖
双击运行 "安装AI依赖.bat"
```
或手动安装：
```bash
pip install ultralytics torch torchvision pyautogui pywin32
```

## 🐛 常见问题

### Q1: Mica 效果选项是灰色的？
**A**: Mica 效果仅在 Windows 11 上可用。

### Q2: 修改缩放后没有变化？
**A**: 界面缩放需要重启应用才能生效。

### Q3: 如何恢复默认设置？
**A**: 删除 `config.json` 文件，重新启动应用即可。

### Q4: 出现警告信息怎么办？
**A**: 亚克力效果的警告不影响使用，可忽略或安装完整版解决。

### Q5: 功能模块打开后显示"功能正在实现中"？
**A**: 该功能的具体业务逻辑还在开发中，可以前往"数据处理"页面使用完整功能。

### Q6: 搜索框搜索不到功能？
**A**: 确保输入的关键词与功能名称相关，支持拼音首字母搜索。

### Q7: 缺少核心依赖无法启动？
**A**: 运行 `python check_env.py` 检查环境，然后双击 `安装核心依赖.bat` 安装缺失的包。

### Q8: YOLO工具箱无法使用？
**A**: 需要安装AI依赖包，请运行 `安装AI依赖.bat` 或查看环境配置说明。

### Q9: 遥感影像识别总是崩溃？
**A**: 这是CPU模式下的固有限制，请查看[遥感影像识别CPU模式崩溃临时解决方案.md](docs/fixes/遥感影像识别CPU模式崩溃临时解决方案.md)了解详细解决方案，强烈推荐使用GPU模式。

更多问题请查看 [使用指南](docs/user/使用指南.md) 或 [环境配置说明](环境配置说明.md)
